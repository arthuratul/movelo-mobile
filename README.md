# Movelo Mobile

Android app for the Movelo delivery platform — built with Expo, React Native, and NativeWind.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Framework | Expo SDK 56 |
| UI Library | React Native 0.85 / React 19 |
| Routing | Expo Router v4 (file-based) |
| Styling | NativeWind v4 (Tailwind CSS) |
| Icons | Lucide React Native |
| Language | TypeScript 6 |
| Font | Inter (via `@expo-google-fonts/inter`) |

---

## Prerequisites

- Node.js 20+
- npm 10+
- Android SDK (for local builds) — or just use the prebuilt APK
- A running instance of the [Movelo backend](https://github.com/your-org/movelo-backend)

---

## Environment Variables

Create a `.env` file in the project root:

```env
EXPO_PUBLIC_API_URL=https://your-backend-url.ngrok-free.app
EXPO_PUBLIC_OAUTH_CLIENT_ID=your-oauth-client-id
EXPO_PUBLIC_OAUTH_REDIRECT_URI=movelo://callback
```

> `EXPO_PUBLIC_*` variables are baked into the JS bundle at build time by Babel. Changing `.env` requires a rebuild.

## Getting Started

```bash
# Install dependencies
npm install

# Start Metro dev server
npm start

# Run on Android (requires emulator or connected device)
npm run android
```

---

## Building an APK

```bash
cd android && ./gradlew assembleDebug
```

Output: `android/app/build/outputs/apk/debug/app-debug.apk`

> The build automatically runs `expo export:embed --reset-cache` so `.env` values are always picked up fresh.

### Regenerating native folders

The `android/` folder is gitignored — it is generated by Expo's prebuild step. If you need to regenerate it:

```bash
npx expo prebuild --platform android
```

After regenerating, re-apply this change to `android/app/build.gradle` inside the `react {}` block:

```groovy
extraPackagerArgs = ["--reset-cache"]
```

## Authentication

Auth uses **OAuth 2.0 Authorization Code flow with PKCE** (RFC 8252) against the Movelo backend auth server. The app never collects credentials — the backend serves all login and signup forms in the system browser.

### Login flow

1. User taps **Sign in with Email** or **Continue with Google**.
2. `initiateLogin()` / `loginWithGoogle()` generates a PKCE code verifier + challenge and opens the backend auth server in the system browser via `expo-web-browser`.
3. The backend handles credentials and redirects to `movelo://callback?code=...`.
4. On Android, `app/callback.tsx` calls `WebBrowser.maybeCompleteAuthSession()` to hand the URL back to the waiting `openAuthSessionAsync` call.
5. The code is exchanged for tokens via `POST /auth/token`.
6. Tokens are stored in `expo-secure-store` and the user is navigated to the dashboard.

### Signup flow

1. User taps **Create account**.
2. `initiateSignup()` opens the backend signup form in the system browser.
3. After the user submits, the backend redirects back to `movelo://callback` and the browser closes.
4. The app shows a "check your inbox" screen.

### Auth endpoints (all under `EXPO_PUBLIC_API_URL/auth`)

| Endpoint | Purpose |
|---|---|
| `GET /auth/authorize` | Backend-hosted login form |
| `GET /auth/google` | Google OAuth initiation |
| `GET /auth/signup` | Backend-hosted signup form |
| `POST /auth/token` | Exchange authorization code for tokens |
| `POST /auth/refresh` | Refresh access token |
| `POST /auth/logout` | Revoke refresh token |

### Token helpers (`lib/auth.ts`)

| Export | Purpose |
|---|---|
| `initiateLogin()` | Open backend login form in browser (PKCE) |
| `loginWithGoogle()` | Open Google OAuth in browser (PKCE) |
| `initiateSignup()` | Open backend signup form in browser |
| `logout()` | Revoke refresh token and clear secure storage |
| `getValidAccessToken()` | Return valid access token, auto-refreshing if needed |
| `getAuthUser()` | Decode JWT and return `{ userId, email }` |

## Design System

Follows the same design tokens as the web frontend — see `DESIGN_SYSTEM.md` (shared with `movelo-frontend`).

- **Primary color:** `#FF6B00` (orange) — use only for the main CTA per screen
- **Typography:** Inter font loaded via `@expo-google-fonts/inter`
- **Touch targets:** minimum `h-11` (44px) on all interactive elements
- **Mobile-first:** NativeWind classes, no web-specific breakpoints
